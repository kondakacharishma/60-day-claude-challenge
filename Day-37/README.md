# Day 37 – Task Compass

## 📌 Challenge Overview

Today, I built an interactive **Task Compass** application using **Claude AI**, **HTML**, **CSS**, and **JavaScript**. 
This management simulation teaches how work flows through real organizations by focusing on ownership, delegation, collaboration, and workflow thinking. 
Through realistic workplace scenarios, users learn how different roles work together to solve problems instead of viewing responsibilities in isolation.

---

## 🎯 Objectives

- Understand how tasks move through an organization.
- Learn the concepts of ownership, delegation, and collaboration.
- Explore communication and workflow across different teams.
- Build a responsive, single-file HTML application that works completely offline.
- Create an engaging learning experience using Vanilla JavaScript.

---

## 🛠️ Features

- 🎨 Premium dark glassmorphism interface
- 🏢 Multiple workplace types (Tech Company, Startup, Corporate Office, etc.)
- 👤 Stage 1 – Who Owns This?
- 🔄 Stage 2 – Task Routing
- 🤝 Stage 3 – Collaboration Challenge
- 📊 Organizational Thinking Dashboard
- 📈 Understanding-based progress tracking
- 🎯 Interactive drag-and-drop activities
- 🔄 Replay with different workplace environments
- 💻 Fully offline single-file HTML application

---

## 📚 What I Learned

- Learned that workplace problems are often solved through collaboration rather than by one individual.
- Understood the importance of assigning clear ownership while involving supporting teams when needed.
- Explored how tasks flow between departments to improve communication and efficiency.
- Learned how delegation and escalation help organizations manage complex situations.
- Improved my understanding of organizational workflows through realistic business scenarios.
- Enhanced my prompt engineering skills using Claude AI.
- Strengthened my frontend development skills using HTML, CSS, and Vanilla JavaScript.
